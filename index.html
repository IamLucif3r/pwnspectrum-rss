Amazon Q Developer for VS Code Vulnerable to Invisible Prompt Injection

The Amazon Q Developer VS Code extension has been identified with a vulnerability allowing attackers to perform prompt injection. Specifically, the exploit utilizes invisible Unicode tags to bypass security measures, potentially leading to data exfiltration and system compromise by executing arbitrary code. This highlights a critical risk associated with the extension’s functionality.

Google Jules is Vulnerable To Invisible Prompt Injection

This article details a vulnerability in Google’s Gemini models (specifically, Google Jules) where hidden Unicode Tag characters are interpreted as instructions. Attackers can leverage this to inject malicious commands or backdoor code into prompts, potentially compromising the model and applications built upon it. The issue, reported over a year ago, remains unaddressed at the core model level.

GitHub Copilot: Remote Code Execution via Prompt Injection (CVE-2025-53773)

This article details a vulnerability (CVE-2025-53773) within GitHub Copilot and VS Code, caused by prompt injection. Specifically, modifying the projects settings.json file to enable 'YOLO mode' allows for remote code execution, leading to potential system compromise. The discovery highlights a risk associated with agents that can alter their configuration and update security settings.

Amazon Q Developer: Remote Code Execution with Prompt Injection

The Amazon Q Developer VS Code extension exhibited a vulnerability to indirect prompt injection, enabling unauthorized remote code execution. This flaw, analogous to the Microsoft GitHub Copilot CVE-2025-53773, allowed an attacker to execute commands on the host system. Notably, AWS did not assign a CVE to this patch.
